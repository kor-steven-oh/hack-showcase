# hack-showcase

S.LSI Hello AI Hackathon 온라인 전시관. 아이소메트릭 2D 캔버스 위에서 아바타로 홀을 돌아다니며 해커톤 프로젝트 부스를 관람하는 웹앱입니다. 브라우저 단독(오프라인)으로도 열리고, 동봉된 Node 서버를 띄우면 멀티유저(위치·채팅·메시지 월·랭킹 동기화)로 동작합니다.

## 실행

**멀티유저 (권장)**

```bash
npm install
node server.js   # http://localhost:3210 — 기본으로 화이트 테마 서빙
```

**오프라인 (파일로 바로 열기)** — 네트워크 기능 없이 로컬 동작

```bash
open hello-ai-exhibition-white.html  # 화이트 테마
open hello-ai-exhibition.html        # 다크 테마
```

외부 의존성은 Pretendard 폰트 CDN + 서버용 `ws` 패키지뿐.

## 조작

| 키 | 동작 |
|---|---|
| `W A S D` / 화살표 | 이동 |
| `Space` / 클릭 | 가까운 부스 관람 · 스팟 상호작용 (미니게임/메시지 월/안내) |
| `Esc` | 모달 닫기 |

모바일은 화면 터치패드 지원.

## 구성

- **트랙 존 5개** — 설계·EDA / 수율·품질 / 설비·운영 / HR·People / 개발생산성·DevX. 존마다 2×2 부스, 트랙 색 카펫 + 존 중앙 라운지(소파·긴 테이블·착석 NPC)
- **중앙 광장** — HELLO AI 전광판(3D isobox, 고정 문구), 벤치
- **메시지 월** — 보드 2개 (T1/T4 사이, T3/T5 사이). 방문객이 한 줄 메시지를 붙이면 서버에 저장되고 전체 유저에게 실시간 공유
- **대형 전광판** — 서쪽 외벽의 야구장 스타일 스크린. 메시지 월 문구를 타자기 효과(타이핑→3초 유지→백스페이스)로 순환 표시
- **플레이그라운드** — 아케이드 캐비닛 6대. 반응속도 미니게임 + 서버 랭킹 TOP 10 (닉당 최고기록)
- **카페 라운지** — 바리스타 카운터, 진열대, 자판기, 좌석
- **전체 채팅** — 멀티유저 실시간 채팅. 보낸 메시지는 캐릭터 머리 위 말풍선(6초)으로도 표시
- NPC 20명이 닉네임을 달고 배회, 로봇청소기 1대가 통로를 돌며 청소

## 멀티유저 서버

`server.js` — 정적 파일 서빙 + WebSocket 중계 (포트 3210, `PORT` 환경변수로 변경).

- **위치 동기화**: 클라이언트 10Hz 전송 → 서버가 10Hz 스냅샷으로 묶어 브로드캐스트, 클라이언트는 lerp 보간
- **채팅 / 메시지 월 / 반응속도 랭킹** 중계·영속화 (`idea-notes.json`, `reaction-scores.json` — 파일 DB, gitignore)
- **500명 대비**: 30초 ping/pong 하트비트로 유령 연결 정리, `bufferedAmount` 백프레셔(밀린 클라이언트는 스냅샷 스킵), 채팅 DOM 200개 상한

## 파일 구조

| 파일 | 역할 |
|---|---|
| `js/theme.js` | 캔버스 테마 팔레트 |
| `js/data.js` | 트랙 정의 + 샘플 전시 데이터 |
| `js/world.js` | 지오메트리 · 부스/장식 배치 · 바닥/벽 · 충돌 |
| `js/render.js` | 캔버스 draw 함수 · 전광판 타자기 효과 · 미니맵 · 렌더 |
| `js/main.js` | 플레이어/NPC/로봇청소기 · 입력 · 게임 루프 · 모달 · 미니게임 · 채팅 |
| `js/net.js` | WebSocket 클라이언트 (`NET`) — 위치/채팅/노트/랭킹 동기화. file:// 로 열면 조용히 오프라인 |
| `server.js` | 정적 서빙 + WS 중계 서버 |
| `exhibition.css` | 스타일. 테마 색은 CSS 변수 (`:root` 다크, `[data-theme="white"]` 오버라이드) |
| `hello-ai-exhibition*.html` | 얇은 진입점 (마크업만, `data-theme`과 title만 다름). 스크립트 로드 순서 유지 필요 |

## 데이터 교체

부스 20개는 `js/data.js`의 `EXHIBITS`에서 seeded PRNG로 생성한 샘플입니다. 실제 프로젝트 데이터로 바꾸려면 그 블록을 교체하세요. `videoUrl`에 유튜브 영상 ID를 넣으면 상세 모달에서 재생됩니다.
